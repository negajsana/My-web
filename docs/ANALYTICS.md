# Аналитика сайта — документация

## Обзор

Самописная система аналитики без сторонних сервисов. Хранит данные о визитах в Neon Postgres (Vercel). Доступна через защищённую паролем страницу.

---

## Структура файлов

```
middleware.ts                        — перехват запросов и трекинг визитов
app/
  stats-9x4kw/
    page.tsx                         — серверная часть страницы статистики
    stats-client.tsx                 — клиентский UI (логин, дашборд, графики)
  api/
    track/
      route.ts                       — API endpoint для записи визита в БД
    stats-auth/
      route.ts                       — API endpoint для авторизации (вход/выход)
    stats-data/
      route.ts                       — API endpoint для получения данных статистики
components/
  language-select-screen.tsx         — изменён: не редиректит на /stats-* страницах
```

## Как это работает

### Трекинг визитов

1. Пользователь заходит на любую страницу сайта
2. `middleware.ts` перехватывает запрос
3. Если страница входит в список отслеживаемых — middleware асинхронно отправляет POST на `/api/track`
4. `/api/track` записывает визит в таблицу `page_views` в базе данных
5. Трекинг не блокирует загрузку страницы — выполняется в фоне

**Отслеживаемые страницы:**
```
/            — Главная
/about       — О нас
/contact     — Контакты
/how-we-work — Как мы работаем
/projects    — Проекты
/services    — Услуги
```

**Что НЕ трекается:**
- Страница статистики (`/stats-*`)
- API маршруты (`/api/*`)
- Статические файлы (`_next`, изображения и т.д.)

### Хранение данных

Таблица `page_views` создаётся автоматически при первом визите.

```sql
CREATE TABLE page_views (
  id         SERIAL PRIMARY KEY,
  slug       TEXT        NOT NULL,   -- путь страницы, например /about
  ip_hash    TEXT        NOT NULL,   -- хэш IP (сырой IP не хранится)
  user_agent TEXT,                   -- браузер/устройство
  referrer   TEXT,                   -- откуда пришёл пользователь
  viewed_at  TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
```

> IP-адреса не хранятся в явном виде — только числовой хэш для подсчёта уникальных посетителей.

### Страница статистики

URL: `https://codearchitect.site/stats-9x4kw`

- Страница не индексируется поисковиками (`robots: noindex`)
- Защищена паролем (cookie-сессия на 7 дней)
- Показывает: общее число визитов и уникальных посетителей, график по дням (последние 30 дней), просмотры по страницам
- При клике на день в графике — детальная статистика за этот день

## API endpoints

### `POST /api/track`
Записывает визит в базу данных.

**Тело запроса:**
```json
{
  "slug": "/about",
  "ip": "1.2.3.4",
  "userAgent": "Mozilla/5.0...",
  "referrer": "https://google.com"
}
```

**Ответ:** `{ "ok": true }`

Ошибки не возвращаются пользователю — трекинг не должен ломать сайт.

---

### `POST /api/stats-auth`
Авторизация. Проверяет пароль и устанавливает cookie.

**Тело запроса:**
```json
{ "password": "..." }
```

**Ответ:** `{ "ok": true }` или `{ "ok": false, "error": "Неверный пароль" }`

---

### `DELETE /api/stats-auth`
Выход. Удаляет cookie сессии.

---

### `GET /api/stats-data?mode=overview`
Возвращает общую статистику.

**Ответ:**
```json
{
  "totalVisitors": 142,
  "totalViews": 380,
  "byPage": [
    { "slug": "/", "views": 200, "visitors": 95 },
    { "slug": "/about", "views": 80, "visitors": 47 }
  ],
  "byDay": [
    { "day": "2026-03-18", "views": 45, "visitors": 20 }
  ]
}
```

---

### `GET /api/stats-data?mode=day&date=2026-03-18`
Возвращает статистику за конкретный день.

**Ответ:**
```json
{
  "date": "2026-03-18",
  "views": 45,
  "visitors": 20,
  "byPage": [
    { "slug": "/", "views": 25, "visitors": 12 }
  ]
}
```

## SEO

Система аналитики не влияет на SEO сайта:

- Страница `/stats-9x4kw` исключена из индексации через `robots: noindex`
- Трекинг выполняется асинхронно и не замедляет загрузку страниц
- Компонент `LanguageSelectScreen` не срабатывает на `/stats-*` маршрутах

---

## Безопасность

- Страница статистики доступна только по секретному URL + паролю
- Cookie сессии: `httpOnly`, `secure`, `sameSite: strict`, срок 7 дней
- IP-адреса хешируются перед сохранением
- Все запросы к `/api/stats-data` проверяют наличие валидной cookie

## Добавление новых страниц для трекинга

Открой `middleware.ts` и добавь slug в массив `TRACKED_SLUGS`:

```ts
const TRACKED_SLUGS = ["/", "/about", "/contact", "/how-we-work", "/projects", "/services", "/новая-страница"]
```

Также добавь читаемое название в `stats-client.tsx` в объект `PAGE_NAMES`:

```ts
const PAGE_NAMES: Record<string, string> = {
  "/новая-страница": "Новая страница",
  // ...
}
```
